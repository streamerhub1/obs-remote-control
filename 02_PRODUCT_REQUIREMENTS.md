# Требования продукта

## Авторизация

- только Twitch OAuth;
- одна учётная запись для сайта и приложения;
- несколько устройств;
- отзыв конкретного устройства;
- завершение всех сессий;
- удаление аккаунта;
- refresh token недоступен renderer-процессу.

## Профиль

- Twitch avatar;
- login;
- display name;
- индивидуальный код;
- статус подписки;
- устройства;
- модераторы;
- доступные стримеры.

Индивидуальный код:

- уникальный;
- человекочитаемый;
- регистронезависимый;
- не раскрывает ID базы;
- постоянный до ручного сброса;
- защищён от перебора rate limit.

## Модераторы

Стример может:

- добавить по коду;
- подтвердить найденный профиль;
- отправить и отменить приглашение;
- менять разрешения;
- приостанавливать доступ;
- удалять модератора;
- завершать активную сессию.

Модератор может:

- принять или отклонить приглашение;
- видеть доступные OBS;
- видеть свои разрешения;
- подключаться к разрешённому OBS;
- покинуть рабочую область.

## OBS

Нужно поддержать доступные через `obs-websocket` функции:

- scenes и scene collections;
- profiles;
- scene items и groups;
- inputs и input settings;
- transforms;
- audio;
- filters;
- transitions;
- studio mode;
- stream;
- record;
- replay buffer;
- virtual camera;
- screenshots;
- statistics;
- hotkeys;
- vendor requests по allowlist;
- все события состояния.

## Desktop

Разделы:

- Главная;
- Мой OBS;
- Доступные OBS;
- Модераторы;
- Профиль;
- Подписка;
- Настройки.

Дополнительно:

- tray;
- autostart;
- автообновления;
- deep links;
- secure storage;
- уведомления;
- экспорт диагностических логов.

## Подписка

- принадлежит стримеру;
- модератор не платит;
- приложение показывает статус;
- управление оплатой открывается на сайте;
- доступ определяется server entitlements;
- поддерживается grace period.
